# 바이브 코딩 사용 가이드

대상: 한국어로 일반적인 프로덕트 개발을 하는 사용자
목표: `p1zza-agent`를 내 제품 프로젝트에 붙여서 쉽게 기능 개발, 수정, 리뷰, 릴리스를 진행하기

## 이 서비스는 언제 쓰나요?

`p1zza-agent`는 `p1zza-agent` 저장소 자체를 관리할 때만 쓰는 도구가 아닙니다.

보통은 이런 상황에서 씁니다.

- 웹 서비스 기능을 하나 추가하고 싶을 때
- 로그인, 결제, API 같은 기능을 고치고 싶을 때
- 디자인과 개발을 역할별로 나눠서 진행하고 싶을 때
- 리뷰, PR, 릴리스까지 한 흐름으로 맡기고 싶을 때

즉, 내 제품 프로젝트 안에서 AI 에이전트 팀처럼 일하게 만드는 서비스라고 보면 됩니다.

## 가장 쉬운 사용 흐름

### 1. 내 프로젝트에서 설치하기

`p1zza-agent`를 내 프로젝트 작업 환경에 설치합니다.

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/zeztto/p1zza-agent/main/install.sh)"
```

또는 clone 후 실행해도 됩니다.

```bash
git clone https://github.com/zeztto/p1zza-agent.git
cd p1zza-agent
./install.sh
```

## 2. 내 프로젝트에서 하고 싶은 일을 바로 말하기

좋은 요청 예시:

- `회원가입 화면을 더 쉽게 바꿔주세요.`
- `랜딩 페이지 전환율을 높일 수 있게 첫 화면을 개선합시다.`
- `주문 API 에러를 고치고 테스트까지 진행하세요.`
- `이번 배포 전에 코드리뷰와 QA까지 진행하세요.`

핵심은 `내 제품에서 무엇을 바꾸고 싶은지` 말하는 것입니다.

## 3. 역할을 나눠서 진행하기

이 서비스는 일을 역할별로 나눠서 처리합니다.

예를 들면:

- 요구사항 정리: `product-manager`, `product-owner`
- 사용 흐름 설계: `ui-ux-designer`
- 화면 구현: `frontend-developer`, `frontend-designer`
- 서버/API 수정: `backend-developer`
- 배포/환경 설정: `infra`
- 검증: `qa`

사용자가 이 역할을 다 외울 필요는 없습니다.
하고 싶은 일을 말하면 시스템이 맞는 역할로 라우팅합니다.

## 자주 쓰는 요청 예시

### 기능 추가

```text
결제 완료 후 주문 추적 페이지로 자연스럽게 이어지도록 개선합시다.
필요하면 UX 정리, 프론트 구현, 리뷰까지 진행하세요.
```

### 버그 수정

```text
모바일에서 로그인 버튼이 눌리지 않는 문제를 고치세요.
원인 분석, 수정, 리뷰까지 진행하세요.
```

### 문서 포함 릴리스

```text
0.8.0 버전으로 이 기능을 반영합시다.
README.md와 CHANGELOG.md도 같이 업데이트하세요.
```

### 코드리뷰 요청

```text
현재 변경사항을 코드리뷰하세요.
```

### 배포 직전 점검

```text
머지 전에 리뷰와 QA를 먼저 진행하세요.
```

## 쉽게 이해하는 핵심 개념

### role

각 에이전트의 담당 분야입니다. 누구에게 어떤 일을 맡길지 정해 주는 개념입니다.

### packet

에이전트끼리 주고받는 정리된 형식의 메시지입니다. 길게 설명하지 않아도 흐름이 덜 꼬이게 도와줍니다.

### review

바로 머지하지 않고, 문제나 빠진 부분이 없는지 먼저 확인하는 단계입니다.

### runtime sync

설정이나 agent 변경을 실제 내 로컬 `~/.claude/`, `~/.codex/` 환경에 반영하는 단계입니다.

## 잘 쓰는 팁

- 프로젝트 이름보다 `무엇을 바꾸고 싶은지`를 먼저 말하면 좋습니다.
- `리뷰까지`, `QA까지`, `머지까지`처럼 끝 범위를 같이 말하면 더 정확합니다.
- 버전 작업이면 `0.x.y` 같은 버전을 먼저 적어주면 문서와 로그까지 같이 맞추기 쉽습니다.
- 사용자에게 보이는 변화라면 `README.md와 CHANGELOG.md도 같이`라고 붙이면 누락이 줄어듭니다.

## 시작할 때 바로 쓸 수 있는 문장

```text
우리 서비스의 회원가입 흐름을 더 쉽게 바꾸고 싶습니다.
필요하면 UX 정리, 프론트 수정, 리뷰까지 진행하세요.
```

```text
0.5.2 버전으로 결제 오류 수정 사항을 반영합시다.
CHANGELOG.md와 README.md도 같이 업데이트하세요.
```
